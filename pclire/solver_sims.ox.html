 solver_sims.ox

Solver implementing the Sims/Lubik and Schorfheide method. This solver solves models of the form $$\sum_{i=0}^k M_{i0} y_{t-i} + \sum_{j=1}^l M_{0j} y_{t+j|t} + \sum_{j=1}^{l} M_{1,j} y_{t+j-1|t-1} = C + Qz_t,$$ where \(y_t\) is \(\mathcal{F}_t\) measurable, but not \(\mathcal{F}_{t-1}\) measurable, i.e. \(y_t\) is non-predetermined. Solutions look like $$y_t + \sum_{i=1}^k M_{01} y_{t-i} = c_s + Q_s z_t.$$

LiRE_SimsSolver :: PerformSolve ( const model )
Solve the canonical form
Parameters:
model The model to solve. This model has to be in canonical form, i.e. in the form \(M_{00} y_t + M_{10} y_{t+1|t} = Q z_t\).
Comments:
The canonical form model is solved by transforming it into the Sims canonical form. This is an internal method. You are advised to use LiRE_Solver::Solve in programs.

LiRE_SimsSolver :: SolveCanonical ( const model , const Pi )
Solve the canonical form \(\Gamma_0 y_t = \Gamma_1 y_{t-1} + c + \Psi \varepsilon_t + \Pi \eta_t\).
Parameters:
Gamma0 an \(m \times m\) matrix
Gamma1 an \(m \times m\) matrix
c an \(m\) column vector
Psi an \(m \times l\) matrix
Pi an \(m \times k\) matrix
Returns:
a solution in the form a LiRE_Solution object or, in case of an error, a LiRE_SolveError object containing an error message and roots if available
Comments:
The solution method is based on Sims (2000) and Lubik & Schorfheide (2003).
