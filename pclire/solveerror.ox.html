 LiRE_SolveError

The LiRE_SolveError class represents information about an error that occurred while solving a model. It is returned by solvers. Solvers only return LiRE_SolveError when they are unable to return a solution. For example, the LiRE_GenericSolver can not handle indeterminacy and will return a LiRE_SolveError object with code SOLVE_ERROR_NOT_UNIQUE. It is still possible to read the model's roots.
Example:
The following code demonstrates the use of LiRE_SolveError 
decl LiRE = new LiRE(2,2);
...
decl result = LiRE.Solve();
if (isSolveError(result)) {
   if (result.Code() == SOLVE_ERROR_NO_SOLUTION) {
      println("The model has no solution!");
   }
   println(result.Message());
   exit(0);
}
else {
   decl solution = result;
   ...
}

 ErrorCode

LiRE_SolveError :: ErrorCode ( )
Returns the error code. The error code can have one of the following values, or a custom value defined by specific solvers:

SOLVE_ERROR_NO_SOLUTIONNo solution exists
SOLVE_ERROR_NOT_UNIQUEThe solution is not unique
SOLVE_ERROR_SCHUR_FAILEDThe Schur decomposition failed

 Print

LiRE_SolveError :: Print ( )
Print the error message and exit the program. This is useful for shorter programs in which the user is only interested in setting up a model, solving it, and printing the solution.
Example:
The following code demonstrates the use of Print() 
decl LiRE = new LiRE(2,2);
...
decl result = LiRE.Solve();
result.Print();
Note that this is in fact a case of abuse of language: the Print() method refers to either the Print method, or the Print method, depending on whether an error occurred or not. In any case, if the solver returns a solution, this program will print the solution. If an error occurred, it will print the error message and exit.
